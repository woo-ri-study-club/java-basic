패키지는 기능에 따라 클래스를 묶을 수 있는 계층 구조를 이루는 포장 용기이다.

### 패키지 사용 시 방법, 규칙
1. 패키지가 다르다면 동명의 클래스 생성 가능
2. 패키지의 이름과 위치를 폴더 위치와 동일하게
3. 패키지는 모두 소문자로 작성
4. 실무에서는 회사의 도메인을 뒤집어서 작성

### 패키지 사용 시 장점
1. 내가 개발한 프로그램의 관리가 용이하다.
2. 다른 패키지의 기능을 import 해 이용 가능
